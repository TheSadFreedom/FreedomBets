# 1. Обзор и запуск

## 1.1. Что такое FreedomBets

FreedomBets — это **личный трекер ставок** на матчи киберспорта. Приложение не подключается к букмекерским конторам и не делает реальных ставок. Вы вручную вносите:

- на какую команду поставили;
- сумму и коэффициент (кэф);
- результат: ожидание / выигрыш / проигрыш.

На основе этого считаются **баланс**, **винрейт**, **профит**, статистика по командам и турнирам.

### Основные возможности

| Возможность | Кратко |
|-------------|--------|
| **Профили** | Несколько «игроков» на одном компьютере, у каждого свой баланс и история |
| **Ставки** | Матч, карта, пистолетный раунд; форматы BO1/BO3/BO5 |
| **Матчи** | Расписание матчей; под матчем видны ставки **всех** профилей |
| **История** | Таблица ставок текущего профиля с действиями WIN/LOSE/отмена |
| **Статистика** | Сводка по ставкам профиля |
| **Топ** | Рейтинг профилей по профиту |
| **Команды** | Статистика по командам по **всем** ставкам |
| **Турниры / Major** | Группировка ставок по турнирам |
| **Pick'em** | Скриншоты стадий Major + медали профиля |
| **Admin** | Создание матчей/турниров, редактирование (профиль с `role: "admin"` или именем `admin`) |

## 1.2. Глоссарий (словарь)

Если вы новичок в проекте или в ставках — начните отсюда.

| Термин | Значение в FreedomBets |
|--------|------------------------|
| **Профиль** | Учётная запись без пароля. Имеет имя, баланс, винрейт. Выбирается на стартовом экране |
| **Ставка (bet)** | Запись: команды, сумма, кэф, тип рынка, статус |
| **Кэф (odds)** | Множитель выплаты. Ставка 100 ₽ при кэфе 2.0 → при WIN на баланс +200 ₽ |
| **Баланс** | Виртуальные деньги профиля. Уменьшается при создании ставки, меняется при расчёте |
| **Статус ставки** | `WAIT` — в игре, `WIN` — выигрыш, `LOSE` — проигрыш |
| **Рынок (betMarket)** | `match` — победа в матче, `map` — на карте, `pistol` — пистолетный раунд |
| **Формат** | `BO1`, `BO3`, `BO5` — сколько карт максимум в серии |
| **Турнир (event)** | Организатор (IEM, CCT…) + название |
| **Tier турнира** | `Major`, `Big`, `Small` — уровень турнира |
| **Major stage** | Стадия Major: Stage 1, Stage 2, Stage 3, Playoff |
| **Матч (match)** | Запись о конкретной игре двух команд в рамках турнира (не привязана к профилю) |
| **Pick'em** | Блок скриншотов прогнозов на стадии Major (привязан к профилю) |
| **Медаль** | Картинка-награда профиля (base64 в базе) |
| **db.json** | Файл-база данных на диске |
| **API** | HTTP-интерфейс для чтения/записи `db.json` |
| **Vite** | Сборщик и dev-сервер для React |
| **json-server** | Библиотека, которая из JSON-файла делает REST API |

## 1.4. Установка Node.js (Windows)

1. Скачайте LTS с [nodejs.org](https://nodejs.org/).
2. Установите с настройками по умолчанию.
3. Откройте **PowerShell** или **cmd**.
4. Перейдите в папку проекта:

```powershell
cd D:\Projects\FreedomBets
```

(замените путь на свой)

5. Установите зависимости **один раз**:

```powershell
npm install
```

В папке появится `node_modules/` — это библиотеки проекта, в git обычно не коммитят.

## 1.5. Первый запуск (пошагово)

### Шаг 1. Запустите API

```powershell
npm run server
```

Ожидаемый вывод:

```
FreedomBets API started on PORT :3001
Watching db.json...
Endpoints:
http://localhost:3001/profiles
http://localhost:3001/bets
...
```

Окно **не закрывайте** — пока оно работает, API доступен.

### Шаг 2. Запустите фронтенд (второй терминал)

```powershell
npm run dev
```

Ожидаемый вывод:

```
VITE v7.x.x  ready in ... ms
➜  Local:   http://localhost:5173/
```

### Шаг 3. Откройте браузер

Перейдите на **http://localhost:5173/**.

### Шаг 4. Выберите или создайте профиль

- На экране «Профили» нажмите на существующий профиль **или**
- Введите имя внизу и нажмите «Создать».

Пароля нет. Выбранный профиль запоминается в `localStorage` браузера (ключ `freedombets.activeProfileId`).

## 1.6. Что вы увидите после входа

```
┌─────────────────────────────────────────────────────────┐
│  [HLTV] [BetBoom]     [LOGO]              [Профиль ▼]   │  ← шапка
├─────────────────────────────────────────────────────────┤
│  [Матчи] [История] [Статистика] [Топ] ...               │  ← вкладки
├─────────────────────────────────────────────────────────┤
│                                                         │
│              содержимое выбранной вкладки               │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

- **Шапка слева** — ссылки на HLTV и BetBoom.
- **Логотип по центру** — пасхалка (случайное сообщение при клике).
- **Справа** — имя профиля, баланс, настройки.

У профиля **admin** (или с `"role": "admin"` в `db.json`) появятся кнопки «Новый матч» и «Новый турнир».

## 1.7. Как остановить

В каждом терминале нажмите **Ctrl+C**.

Данные сохраняются в `db.json` автоматически при каждом изменении через API.

---

## 1.8. Production-сборка (опционально)

```powershell
npm run build
npm run preview
```

- `build` создаёт статические файлы в `dist/`.
- `preview` отдаёт их локально.

Для полноценной работы **всё равно нужен API** (`npm run server`), потому что данные в `db.json`, а не в собранном фронтенде.

---

## 1.9. Порты

| Сервис | Порт по умолчанию | Назначение |
|--------|-------------------|------------|
| Vite (dev) | 5173 | React-приложение |
| API | 3001 | REST + статика `public/` |

Если 5173 занят, Vite выберет 5174, 5175… — смотрите вывод в терминале.

Порт API можно сменить:

```powershell
node server/index.mjs db.json --port 4000
```

Тогда в `vite.config.ts` нужно поменять `target` в `server.proxy`.

## 1.10. Десктоп-приложение (кратко)

Windows-версия на Electron — одно окно, API и React внутри.

```powershell
npm run desktop:dev    # разработка
npm run desktop:dist   # локальный .exe в release/
```

Данные — в `%AppData%\Roaming\FreedomBets\`, не в папке установки.
